Tech Blog #1 - CSS styling

Inline vs Inline-block 16 Sept 2022



I've done it! I have dipped my toes in to the waters of styling my webpage using CSS (cascading style sheets). This is what I have learned about Inline and Inline-block properties.

When applying CSS styling to your webpage, Inline and Inline-block are two display property values that states how an element (e.g. title, paragraph, picture) should be aligned on your webpage.
This is an example of what inline, inline block and block outputs look like



What is inline?

display: inline;

There are some elements you may already be using that are inherently inline such as: Span, Anchor and Bold; which style text sitting along the same line/s. As shown in the image above, inline elements will share the same line with another element. They will take up as much width space as they need and won't change their size. Any height and width settings you apply will be ignored. Here is an example of working around the height and width restrictions that you can face with the display: inline property:

Because text is inline by default, we dont need to use the display: inline property at all. Instead, the background for each element was styled by changing the background, text color and adjusting margin, height and size properties.


What is inline-block?

display: inline-block;

Just like the 'inline' display property above, elements with the inline-block display property can sit on the same line as other elements, however, a key difference is that inline-blocks will accept your width and height settings. They are squares that share the space.
This comes in handy when you want to create a navigation bar with your links sitting in blocks beside each other, or, you want to sit two elements next to each other (e.g. two text boxes).

Here is an example of what the HTML structure, CSS styling and their output looks like:

The screen shot above shows the HTML structure and CSS styling for the three purple elements at the bottom. The contents within the boxes are styled using inline display properties (span and small); the elements themselves are displayed using inline-block. Basically the boxes sit next to each other in a line.


What is block?

display: block;

Block is the opposite of inline. They are squares that dont share (space that is). When you use the display property 'block', the entire width of the page is reserved for that element. It wont let you place any other HTML elements next to it unless they have the 'float' property. There are some elements that you may already be using which are inherently block styled such as: paragraphs divs, headings. All other elements inline with paragraphs, headings and divs will shift to the next line.



Explained in non-tech language

Imagine your elements on a webpage as luggage bags. Each bag has one of three styles: a hard shell bag with wheels, a soft duffle bag, and a boxes labeled 'Fragile'.
The fragile bags are elements using the block display property and they wont share the space with any other bag (element).
The soft duffel bags can be placed on the conveyor belt with other items and can flex and bend to fit in the width of the space however you need.
The hard shell bags are not flexible at all and you can only shift them left or right to fit other bags on the conveyor belt with them.
