What is Control Flow?

JavaScript will normally read and follow code instructions from the top down, and from the left to the right of each line. Just like you would do reading a book (here in NZ that is). This is JavaScripts default flow.
You can change (or control) the direction of this flow by using conditions and loops. Applying a loop and using conditions such as if statements, switch, and ternary can instruct JavaScript to divert its flow when criteria are met, and even loop through a section of code repeatedly until criteria are met.

Like a bus travelling the same route every day. It starts at the bus depot, makes its way through the selected streets, and at the end of the day it returns to the bus depot. This is the bus's default flow.
After leaving the depot every morning, the bus is given a set of streets to loop (or cycle) through until a condition is met e.g. the drivers shift has ended or it has completed all its scheduled stops for the day.



What is the DOM?

"[The DOM] is a platform and language-neutral interface that allows programmes and scripts to dynamically access and update the content, structure, and style of a document."
(w3schools.com)

DOM stands for Document Object Model. It is the map of your webpage that helps JavaScript to easily find and edit the contents on your webpage. When a webpage is loaded, the browser creates a DOM of the page and sorts HTML elements in to a tree like hierarchy to help make sense of everything. The individual objects of the tree structure are sorted in to parent branches and/or children branches. These are referred to as 'nodes'.


With the documents' structure and content clearly outlined, the DOM gives you the ability to interact with objects in the HTML document. You can use code (like JavaScript) to query (pick out) objects or elements by CSS classes, Id's etc. and manipulate them by changing their style (attribute), or text values (innerHTML) just to name a couple.



Accessing data from Objects and Arrays - whats the difference?

What is an object?

Objects represent a collection of items which can be defined by a set of characteristics (properties) and key:value pairs. Objects are used to group together like data using a single data structure, housed in curly braces '{ properties, key names and values in here }'.


e.g. This is an unordered collection of menu items at a cafe. The list of products above are stored as an object. Inside the object are different properties (e.g. whiteCoffee or blackCoffee) with related key names and values (e.g. how many are in stock and their prices).
Using an array in our functions will easily and accurately find, display or manipulate any of the values inside of the object.


What is an array?

Arrays are used to group a list of items/elements. We first create a single variable (the heading in which they will sit under e.g. products) and list the items or elements inside square brackets separated by a comma:

let products = ["whiteCoffee", "blackCoffee", "muffin", "eggs"]
An array is a simpler form of an object as it doesnt require key:value pairs. Here is where arrays differ slightly from objects. Array items are indexed starting at zero (0) a.k.a zero-indexed.


The difference between accessing data from arrays and objects

When pulling data from an object, you can use dot notation or bracket notation stating the name of the object and the name of the key.

    Dot notation looks like this : Object.key or whiteCoffee.stock

    Bracket notation looks like this : Object["key"] or whiteCoffee["stock"]



This is different to pulling data from an array, because arrays are zero-indexed. Meaning you would replace the name of the key with the index number:
Like this : products[1] which will return "whiteCoffee"


Functions - why they're helpful

A function is a body of statements that define a logic, or, give details for JavaScript to action a task / calculate a result for you. Functions are fundamental to JavaScript because they are reusable, and you dont have to re-write code over and over again. A function can use the likes of Objects, Arrays, Loops and Condition Statements to do the thinking for you and require less writing of code and logic overall. They not only get the job done quickly, but they make processing huge amounts of data faster, easier and accurate.
